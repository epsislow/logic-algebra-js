The Combinational Lookup Table (lut)

The lut component is a zero-delay, combinational data matrix. The moment the input signal changes, the output updates instantly in the exact same execution step without waiting for a clock cycle.Syntax DefinitionUses the data { ... } block to declare address mappings.Supports single address declarations in decimal (\) or hex (^).
Supports Range Selectors (X - Y : value) to bulk-assign multiple addresses to a single constant output.Unmapped addresses automatically default to all zeros 

comp [lut] .decoder:
  depth: 4         # Output bit-width
  length: 16       # Total addressable slots
  = data {
    0           : 0001   # Address 0 outputs 0001
    \1 - \5     : 0010   # Addresses 1, 2, 3, 4, and 5 all output 0010
    ^a - ^f     : 1111   # Hex addresses 0xA through 0xF all output 1111
    # All skipped addresses (6, 7, 8, 9) automatically output 0000
  }
  :
  
Use code with caution.How to Use the LUTBecause it is fully combinational, you completely bypass sequential property blocks (:{ at = X }). You read and write directly on the main circuit board via inline function mappings or raw pin assignments.

# Method A: Direct Inline Functional Mapping (Cleanest)

4wire current_state = .decoder(in = current_index) # 'current_state' updates instantly

# Method B: Structural Wire Assignments
.decoder:in = .my_counter:get
4wire current_state = .decoder:out

To handle negative numbers and relative jumps (like jumping 3 lines backward or 4 lines forward), the assembler needs to know whether a bit-field should treat numbers as standard unsigned positive numbers, or as signed numbers using Two's Complement.We can add a simple modifier to our bit-width tokens—the letter S for Signed:4b = Unsigned 4-bit field (Values from 0 to 15).S4b = Signed 4-bit field (Values from -8 to +7).RS4b or AS4b = Signed fields that also require a Register (R) or Address (A) prefix.Here is exactly how this works in the definition and under the hood during compilation.1. Defining a Relative Branch InstructionIf you are designing a conditional branch instruction (like BEQ - Branch if Equal), the jump distance is relative to the current position of the Program Counter (PC). You define it with a signed field (S4b):logts-playasm [mini-cpu] .my_isa:
  # 4-bit opcode + 4-bit signed relative jump offset
  BEQ : 0100 + S4b  
  :

2. How the Compiler Calculates the Math (Behind the Scenes)
When a student uses a label with a signed field, the assembler calculates a relative offset instead of a hardcoded absolute address:
\(\text{Offset}=\text{Target\ Label\ Address}-(\text{Current\ Instruction\ Address}+1)\)(Note: We add +1 because by the time the CPU executes an instruction, the Program Counter has already moved to the next line)

.Look at this example:logts-play= .my_isa {
  loop_start:         # Address \0
    NOP               # Address \0
    NOP               # Address \1
    BEQ loop_start    # Address \2 (This instruction!)
  }
Use code with caution.When the compiler hits line 3 (BEQ loop_start at Address 2), it runs the math:Target Address (loop_start) = 0Next Address (Current Address 2 + 1) = 3Math: 0 - 3 = -3The assembler needs to pack the negative number -3 into the S4b field.3. Converting to Two's Complement BinaryTo turn -3 into a 4-bit binary string, the compiler uses standard Two's Complement logic:Take the positive binary version of the number 3 ➔ 0011Invert all the bits (flip 0 to 1 and 1 to 0) ➔ 1100Add 1 to the result ➔ 1101The assembler safely packs 1101 into the instruction template:Final Output Bits: 0100 (Opcode) + 1101 (Offset) ➔ 01001101When the CPU runs this instruction, it takes its current PC (3), adds the signed value -3 (1101), and its PC instantly becomes 0, perfectly looping back to loop_start!4. Catching Student ErrorsIf a student writes a loop that is too far away, the signed bounds check will catch it instantly:logts-play= .my_isa {
  loop_start:
    # ... imagine 20 lines of NOPs here ...
    BEQ loop_start
